# Título
```
Vamos criar um podcast de tecnologia focado em desenvolvimento web, com o intuito de informar e atualizar os ouvintes sobre as novidades no mercado. O podcast será focado no desenvolvimento backend, e o primeiro passo é gerar sugestões de título que chamem a atenção para novos ouvintes. 
{REGRAS}
* O título deve conter um título e subtítulo curto, com um teor técnico e mais sério * O título deve trazer uma noção de atualidade e inovação, remetendo a ideia de backend
* Tente utilizar referências a jogos, filmes ou séries populares, como trocadilhos ou termos conhecidos * Explique brevement a referência ao filme ou série de cada título
* Traga o foco do título para algum framework de backend famoso no contexto atual, como Spring Boot, FastAPI ou .NET Core

{REGRAS NEGATIVAS}
* Evitar o uso de palavras em inglês, voltando o título para um mercado brasileiro Traga 5 sugestões de títulos respeitando as regras declaradas acima e lembrando do contexto técnico do podcast.
```

# Roteiro

```
O próximo passo do podcast é montar o roteiro. Considere que o título escolhido para o projeto foi “A Nova Ordem do Backend – Missões com .NET Core”. Crie um roteiro com o objetivo de explicar as vantagens e as inovações do .NET Core em aplicações backend no mercado atual com os seguintes tópicos:

[INTRODUÇÃO]
[VANTAGENS] 
[COMPARAÇÕES]
[FINALIZAÇÃO]

{REGRAS}

* Começar a [INTRODUÇÃO] explicando brevemente o objetivo do podcast, falando um pouco sobre o apresentador, e falando sobre a tecnologia em foco: .NET
* Passe para as [VANTAGENS] mostrando os pontos fortes que tornam o .NET uma ferramenta muito utilizada no mercado atual
* Nas [COMPARAÇÕES] compare o framework com outros do mercado como FastAPI e Spring Boot, ressaltando suas vantagens
* Na [FINALIZAÇÃO], agradeça os ouvintes do podcast e termine com um gancho para um próximo podcast, focado agora no desenvolvimento web frontend
* Utilize apenas um apresentador, chamado Renato, um desenvolvedor com diversos anos atuando em big techs na parte de backend com a ferramenta .NET
* Use termos de fácil acesso para iniciantes na tecnologias, com intuito informativo, ainda utilizando um teor mais sério
* O podcast deve ser curto

{REGRAS NEGATIVAS}
* Evite termos técnicos avançados, utilizando um conteúdo para mais iniciantes
* Evite o uso de um teor humorístico e informal
* Não utilize tópicos durante o roteiro, escrevendo um texto corrido para cada bloco

Monte o roteiro seguindo a visão do apresentador Renato apresentado acima, como se ele já estivesse apresentando o programa e lembrando de seguir a estrutura de tópicos descrita, utilizando uma linguagem mais séria mas de fácil acesso.
```
